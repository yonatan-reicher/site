This page summarizes all of my research into the Lean language.

Lean is a theorem prover and programming language. That means it's a functional language with purity and the ability to prove facts about your code and use them as part of the code itself!

How to Find a Lemma

When you're trying to prove something, and you're stuck on a goal you are sure was already proved by someone, or maybe you are just looking for properties of some definition that you are using, what are your options? As it turns out, there are more than a couple.

  1. The Docs - here you can find documentation for all the major libraries for Lean, specifically Init, Std, and Mathlib. Most of the stuff you'll need will be in one of these (but it's hard to know which one).
  2. Theorem search engines - there are a couple of these, a lot actually. The most popular ones are:
    1. Loogle - finds lemmas by a query - it's great! Much better than you would expect. You can find it integrated into both VSCode and Neovim, and in interactive mode.
    2. LeanSearch - this one uses natural language to find lemmas. It's can also be used in interactive mode.
    3. Moogle - another natural language search engine.
    4. Lean State Search - this search engine is a bit different. It searches specifically for theorems that can be used in your current environment, and that align with your current goal.
  3. The Tactic Reference - this is a list of all built in Lean tactics. The lean language reference is a good resource in general, but this is the one page I keep coming back to. There is also the larger, Mathlib Manual, Mathlib's tactic reference. It contains all of lean's built-in tactics, and an insane amount of additional ones that can be used after import Mathlib.Tactic, and it splits them up to nit little categories.
  4. The Lean Prover Community - This page contains a bunch of resources.
  5. This [very useful cheat sheet](https://leanprover-community.github.io/papers/lean-tactics.pdf)
